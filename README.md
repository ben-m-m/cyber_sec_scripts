Overview

This project simulates a Secure Vault system. It allows:

Secure storage of secrets and logs

Controlled file permissions

Safe updates to secrets

Monitoring for security risks

The project is divided into four steps, each implemented as a Bash script.

Repository Files
File	Purpose
vault_setup.sh	Initializes the vault directory and files (keys.txt, secrets.txt, logs.txt). Adds a welcome message and lists files.
vault_permissions.sh	Checks and updates file permissions interactively or applies defaults (keys.txt â†’ 600, secrets.txt â†’ 640, logs.txt â†’ 644).
vault_ops.sh	Menu-driven operations: add/update secrets, append logs, and restrict access to keys.
vault_monitor.sh	Monitors vault files: displays name, size, last modified date, and permissions. Flags files with permissions >644. Generates vault_report.txt.
vault_report.txt	Generated by vault_monitor.sh. Contains a report of file metadata and security warnings.
